# Surrogate Studio - Active Plan (Mar 2026)

## 1) Direction (locked)
- Platform is schema-first, module-based, runtime-agnostic.
- `oscillator` is one dataset schema/module among many.
- UI is lab-based: Data Lab / Model Lab / Training Lab.
- Repo strategy: in-place development now, split to new repo before release.
- Runtime family split is explicit:
  - `tfjs` = browser/client baseline family, with optional `tf-node` / `server_tfjs` adapter on server
  - `pytorch` = server/notebook baseline family
- Trainer owns its worker/runtime session until delete/clear/reload.

## 2) Current State

### Done
1. Shared contracts
- schema registry + dataset runtime bridge
- runtime handshake/event normalization
- workspace store contract (dataset/model/session/epoch history)

2. Shared UI engines
- left item list renderer reused across labs
- right inspector form renderer reused across labs

3. Dataset modules
- `oscillator`, `mnist`, `fashion_mnist` registered
- module build path active in Data Lab
- image-source lazy load/cache flow active

4. Model graph contract
- drawflow graph is canonical model structure
- schema-scoped presets
- schema feature-node policy enforced in model feature node controls
- built-in schema presets are declarative (`metadata.graphSpec`)
- model node palette is rendered from schema contract, not hardcoded tab buttons

5. Training/export baseline
- single selected trainer main view is active (no multi-card main panel)
- selected sessions export to notebook bundle from Training Lab

6. Repository split prep
- `REPO_SPLIT_PLAN.md` added
- `scripts/stage_platform_repo.sh` added for pre-release staging copy

### Partial / pending
1. Non-oscillator training path in run queue is not fully enabled yet.
2. Server runtime adapters remain scaffold-level (not full parity).
3. Module custom render override is partial; shared schema-driven render is the primary path.
4. Remaining legacy `app.js` logic still needs to be pushed into shared core modules.
5. Model payload shape must be normalized consistently as `graph + per-node config`.
6. Shared item-panel/config-panel modules are not yet formalized as first-class core modules.
